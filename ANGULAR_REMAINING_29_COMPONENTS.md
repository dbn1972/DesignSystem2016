## 🎯 Component Specifications

Each component should follow Angular 18+ standards:

### Standard Component Structure
```typescript
import { Component, ChangeDetectionStrategy, ViewEncapsulation } from '@angular/core';
import { CommonModule } from '@angular/common';

@Component({
  selector: 'ux4g-component-name',
  standalone: true,
  imports: [CommonModule],
  templateUrl: './component-name.component.html',
  styleUrls: ['./component-name.component.css'],
  changeDetection: ChangeDetectionStrategy.OnPush,
  encapsulation: ViewEncapsulation.None
})
export class ComponentNameComponent {
  // Component logic
}
```

### For Form Components
```typescript
import { ControlValueAccessor, NG_VALUE_ACCESSOR } from '@angular/forms';

@Component({
  // ... other config
  providers: [{
    provide: NG_VALUE_ACCESSOR,
    useExisting: forwardRef(() => ComponentNameComponent),
    multi: true
  }]
})
export class ComponentNameComponent implements ControlValueAccessor {
  writeValue(value: any): void { }
  registerOnChange(fn: any): void { }
  registerOnTouched(fn: any): void { }
  setDisabledState(isDisabled: boolean): void { }
}
```

## 💡 Implementation Tips

### 1. Start Simple
Begin with simpler components (Spacer, Divider, Section) before tackling complex ones (Chatbot, Form Builder)

### 2. Reference React
All 73 components exist in React with working examples - use them as reference

### 3. Government Patterns
Focus on Indian government patterns:
- Aadhaar validation (12 digits)
- PAN format (ABCDE1234F)
- Indian states/districts
- Hindi/English bilingual

### 4. Reuse Existing
Leverage existing Angular components:
- Dropdown might reuse Menu logic
- Modal might extend Dialog
- Feedback Rating could use Rating component

### 5. Batch Work
Build similar components together:
- All layout utilities in one batch
- All government forms together
- All advanced features as a group
